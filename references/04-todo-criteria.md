## 判定标准

一条记录必须**同时满足全部 6 条**才列为待办：

| # | 条件 | 说明 | 判定方法 |
|---|------|------|---------|
| ① | 来源合规 | 来自用户选择的渠道，且在时间范围内 | 自动化判定 |
| ② | 状态未完成 | 截至提取日，该事项未在近7天内闭环完成 | 规则+判断 |
| ③ | 具体工作 | 指向具体工作或任务，非咨询/询问类 | 规则判断 |
| ④ | 责任明确 | 由用户自己完成，或由用户亲自布置给团队成员 | 规则判断 |
| ⑤ | 含动作 | 含明确或隐含的动作要求（做、确认、推进、跟、回复等） | 规则判断 |
| ⑥ | 时间合适 | 应在下周完成（含由近7天顺延的） | 自动化判定 |

## 排除清单

以下情况**不列为待办**：

| 排除类型 | 示例 | 判断依据 |
|---------|------|---------|
| 已完成 | "已提交"、"已发你"、"收到"、"👌"后无追问 | 看后续有无关闭环标记 |
| 咨询/询问 | "这个发哪里？"、"怎么设置？"、"需要确认吗？" | 一句话能决定的，非持续性任务 |
| 非我安排 | 总部群发的通知、同事间互相@的任务 | 不是你布置或指向的团队工作 |
| 信息同步 | "今天总部发布了推文"、"7月排期如下" | 纯信息告知，无动作要求 |
| 待办记录文档 | 飞书"7月工作计划"、钉钉"周待办"文档本身 | 避免循环提取 |

## 分类规则

仅两类，每类按紧急程度排序：

### 类别一：由我完成
排序规则（优先级由高到低）：
1. 有明确截止日且即将到期 → 标记🔥
2. 上级/领导指令 → 排前
3. 本周内完成即可 → 标记🟡
4. 持续推进、无截止日期 → 标记🔵，排最后

### 类别二：由团队完成（我布置的）
排序规则（优先级由高到低）：
1. 有明确截止日且即将到期 → 标记🔥
2. 跟向上汇报相关的 → 排前
3. 本周内完成即可 → 标记🟡
4. 持续推进 → 标记🔵，排最后

**关键规则**：
- 责任人为团队其他人的事项，**必须是由用户亲自布置或指向的**
- 非用户安排的工作（如总部通知、同事间互相@）不列入

## 判定边界案例

以下来自实际实践中的边界判断参考：

| 案例 | 判定 | 理由 |
|------|------|------|
| 同事问"比价报告发哪个群？" | 排除 咨询类 | 一句话回复即可，非持续性任务 |
| 同事问"需要给领导再确认下吗还是可以直接下发？" | 待办 | 卡住了下一步，需要你决策 |
| 同事回了"好的"但未交付成果 | 待办 | 你布置的任务，有明确动作要求，且未确认完成 |
| 同事解释了某个原因 | 排除 咨询类 | 解释已给出，认不认可是一句话的事 |
| 群聊中@其他人处理日常工作 | 排除 非我安排 | 非你布置 |
| "7月排期如下" + 图片 | 排除 信息同步 | 纯告知，无动作要求 |
| "这个方案你再改一下，X号前给我" | 待办 | 有动作要求+截止时间+责任明确 |
